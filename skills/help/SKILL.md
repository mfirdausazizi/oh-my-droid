# Help Skill

Guide users on Oh-My-Droid capabilities and usage.

## What OMD Does Automatically

| When You... | I Automatically... |
|-------------|-------------------|
| Give complex task | Parallelize and delegate to specialists |
| Ask to plan | Start planning interview |
| Need completion | Persist until verified |
| Work on UI | Activate design sensibility |
| Say "stop" | Intelligently stop operation |

## Magic Keywords

| Keyword | Effect |
|---------|--------|
| **ship:** | Primary end-to-end flow |
| **ralph:** | Persistence mode |
| **plan:** | Planning interview |

## Available Commands

| Command | Description |
|---------|-------------|
| `/omd-init` | One-time setup |
| `/omd-ship` | Primary end-to-end flow |
| `/omd-autopilot` | Autonomous execution |
| `/omd-ralph` | Persistence mode |
| `/omd-plan` | Specification mode |
| `/omd-swarm` | Multi-agent coordination |
| `/omd-code-review` | Code review |
| `/omd-security-review` | Security audit |
| `/omd-analyze` | Deep analysis |
| `/omd-doctor` | System diagnostics |
| `/omd-help` | This help |

## Tiered Droids

| Tier | Use For |
|------|---------|
| LOW | Simple lookups, quick tasks |
| MEDIUM | Standard work |
| HIGH | Complex analysis, verification |

## First Time Setup

Just run `/omd-init` - that's the only command you need to know initially.
